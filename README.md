# LanguageTool Offline — Офлайн проверка орфографии для Chrome

Chrome-расширение для проверки орфографии и пунктуации в реальном времени с использованием локальных словарей. Работает полностью офлайн без отправки данных на сервер.

## 📋 Возможности

- ✅ **Офлайн работа** — все проверки выполняются локально
- ✅ **Два языка** — поддержка английского (466K+ слов) и русского (1.5M+ слов)
- ✅ **Проверка в реальном времени** — автоматическая проверка при вводе текста
- ✅ **Подсветка ошибок** — красная двойная линия под ошибочными словами
- ✅ **Подсказки** — tooltip с вариантами замены при клике на ошибку
- ✅ **Контекстное меню** — быстрая замена слова через правый клик
- ✅ **Исключения** — возможность добавлять слова в список исключений
- ✅ **Логирование** — полное логирование всех действий
- ✅ **Тёмная тема** — автоматическое переключение

### Ключевые компоненты

#### 1. Trie (Префиксное дерево)
Оптимизированная структура данных для быстрой проверки слов:
- Проверка существования слова: O(m), где m — длина слова
- Поиск по префиксу для подсказок
- Эффективное использование памяти

#### 2. IndexedDB
Хранение словарей в браузере:
- Массовая загрузка батчами по 10 000 слов
- Сохранение между сессиями
- Асинхронный доступ без блокировки UI

#### 3. Content Script
Работа с текстовыми полями на странице:
- Поддержка `<input>`, `<textarea>`, `[contenteditable]`
- Debounce проверка (300ms задержка)
- Overlay для подсветки ошибок в textarea

#### 4. Background Service Worker
Фоновая обработка запросов:
- Проверка слов через Trie
- Управление контекстным меню
- Синхронизация с storage

## 🚀 Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/isalnikov/LanguageTool.git
cd LanguageTool
```

### 2. Загрузка полных словарей

Для работы с полными словарями скачайте файлы `words.txt`:

**Английский словарь** (466 550 слов):
```bash
# Поместите файл в vocab/en/words.txt
```

**Русский словарь** (1 532 628 слов):
```bash
# Поместите файл в vocab/ru/words.txt
```

### 3. Установка в Chrome

1. Откройте Chrome и перейдите на `chrome://extensions/`
2. Включите **Режим разработчика** (переключатель в правом верхнем углу)
3. Нажмите **Загрузить распакованное расширение**
4. Выберите папку `LanguageTool`
5. Расширение установлено! 🎉

## 📖 Использование

### Проверка текста

1. Начните печатать в любом текстовом поле на странице
2. Ошибочные слова будут подчёркнуты красной двойной линией
3. Кликните на подсвеченное слово для показа подсказок

### Замена слова

**Способ 1 — Tooltip:**
1. Кликните на ошибочное слово
2. Выберите правильный вариант из списка

**Способ 2 — Popup:**
1. Кликните на ошибочное слово
2. Нажмите на кнопку с вариантом замены

**Способ 3 — Контекстное меню:**
1. Выделите ошибочное слово
2. Кликните правой кнопкой мыши
3. Выберите вариант замены

### Добавление в исключения

1. Кликните правой кнопкой на ошибочное слово
2. Выберите **"Пропустить слово"**
3. Слово будет добавлено в список исключений

### Управление исключениями

1. Кликните на иконку расширения
2. Прокрутите до раздела **"Игнорируемые слова"**
3. Удалите ненужные слова крестиком
4. Или очистите весь список кнопкой

## 🔧 Разработка

### Требования

- Node.js 16+
- Chrome 88+ (для Manifest V3)

### Структура модулей

#### `logger.js`
Система логирования с уровнями:
```javascript
import { Logger } from './logger.js';

const logger = new Logger('context');
logger.debug('Отладка');
logger.log('Информация');
logger.warn('Предупреждение');
logger.error('Ошибка');
```

#### `trie.js`
Trie структура данных:
```javascript
import { Trie, DictionaryManager } from './trie.js';

const trie = new Trie();
trie.insert('слово');
trie.has('слово'); // true
trie.findByPrefix('сл', 5); // ['слово', ...]
```

#### `dictionary-db.js`
Работа с IndexedDB:
```javascript
import { dictionaryDB } from './dictionary-db.js';

await dictionaryDB.open();
await dictionaryDB.bulkInsert('english_words', words);
const exists = await dictionaryDB.wordExists('english_words', 'word');
```

### Отладка

1. Откройте `chrome://extensions/`
2. Найдите LanguageTool Offline
3. Нажмите **"service worker"** для открытия DevTools
4. Для content script: откройте DevTools страницы и выберите iframe расширения

### Логирование

Логи сохраняются в `chrome.storage.local`:
```javascript
// Получить логи
const storage = await chrome.storage.local.get(['logs']);
console.log(storage.logs);

// Экспортировать через popup
// Нажмите кнопку "Экспорт логов"
```

## 📊 Производительность

### Оптимизации

1. **Trie в памяти** — быстрая проверка слов после первоначальной загрузки
2. **Debounce проверка** — проверка текста через 300ms после последнего ввода
3. **Батч загрузка** — загрузка словарей порциями по 10 000 слов
4. **Ленивая инициализация** — загрузка при первом использовании

###Benchmark

| Операция | Время |
|----------|-------|
| Проверка слова | < 1ms |
| Поиск подсказок | < 10ms |
| Загрузка словаря (en) | ~2-3 сек |
| Загрузка словаря (ru) | ~5-7 сек |

## 🐛 Известные ограничения

1. **Первая загрузка** — при первой установке словари загружаются несколько секунд
2. **Память** — Trie структура занимает ~50-100MB в памяти
3. **Content editable** — поддержка ограничена для сложных редакторов
4. **SVG текст** — текст внутри SVG не проверяется
